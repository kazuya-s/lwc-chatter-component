# 04: デプロイ・動作確認

## ステータス: 完了（手順書として参照）

## 前提条件

- Phase 1（プロジェクトセットアップ）完了
- Phase 2（Apexコントローラー）完了
- Phase 3（LWCコンポーネント）完了
- Salesforce CLI (`sf`) インストール済み
- Dev Hub 認証済み

## デプロイ手順

### 1. ScratchOrg セットアップ（初回）

```bash
# Dev Hub 認証（未認証の場合）
sf org login web --set-default-dev-hub

# ScratchOrg 作成〜デプロイ〜テスト一括実行
./scripts/setup-scratch-org.sh
```

### 2. ScratchOrg をブラウザで開く

```bash
./scripts/open-org.sh
```

### 3. App Builder でコンポーネントを配置

1. 設定 (Setup) を開く
2. **Lightning App Builder** を検索して開く
3. 既存のホームページを編集、または新規ページを作成
4. 左パネルのコンポーネント一覧から **「Chatter Feed（全文表示）」** を検索
5. ページにドラッグ＆ドロップ
6. **保存** → **有効化**

### 4. モバイルアプリでの確認

1. Salesforce モバイルアプリを起動
2. 配置したページを表示
3. 長文のChatter投稿が **折りたたまれずにすべて表示** されることを確認

## 確認ポイント

| 確認項目 | 期待結果 |
|---------|---------|
| 短い投稿 | そのまま全文表示 |
| 長い投稿（100文字以上） | 折りたたまれずに全文表示 |
| 改行を含む投稿 | 改行が維持されて表示 |
| リッチテキスト（リンク等） | リンクがクリック可能な状態で表示 |
| デスクトップ表示 | 正常に表示される |
| モバイル表示 | 正常に表示される（折りたたみなし） |

## テスト投稿の作成方法

1. ScratchOrg でホームページを開く
2. Chatter タブを開く
3. 200文字以上の長文を投稿する
4. 標準Chatterビューで折りたたまれることを確認
5. カスタムコンポーネントで全文表示されることを確認

## トラブルシューティング

| 問題 | 対処 |
|------|------|
| デプロイエラー | `sf project deploy start --dry-run` で事前検証 |
| テスト失敗 | `sf apex run test --code-coverage` でカバレッジ確認 |
| コンポーネントが表示されない | `js-meta.xml` の targets 設定を確認 |
| Chatterデータが取得できない | Org設定でChatterが有効か確認 |
